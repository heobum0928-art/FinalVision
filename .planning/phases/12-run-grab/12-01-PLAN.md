현재 `Btn_start_Click`은 선택된 노드의 시퀀스/액션을 `mParentWindow.StartSequence(seqID, actID)`로 실행한다. 이 로직 앞에 SimulImagePath 분기를 추가한다:

```csharp
private void Btn_start_Click(object sender, RoutedEventArgs e) {
    if (treeListBox_sequence.SelectedIndex < 0) return;

    //260406 hbk -- D-03: SimulImagePath가 설정된 경우 로드된 이미지로 검사 실행
    if (treeListBox_sequence.SelectedItem is NodeViewModel node
        && node.NodeType == ENodeType.Action) {

        ESequence seqID = node.SequenceID;
        if (seqID == ESequence.Inspection) {
            var seq = SystemHandler.Handle.Sequences[seqID];
            int actIndex = -1;
            for (int i = 0; i < seq.ActionCount; i++) {
                if (seq[i].ID == node.ActionID) { actIndex = i; break; }
            }
            if (actIndex >= 0 && seq[actIndex] is Action_Inspection act) {
                var p = act.Param as InspectionParam;
                if (p != null && !string.IsNullOrEmpty(p.SimulImagePath)
                    && File.Exists(p.SimulImagePath)) {
                    //260406 hbk -- D-04: 선택된 Shot만 RunBlobOnLastGrab 실행
                    if (!SystemHandler.Handle.Sequences.IsIdle) return;
                    act.RunBlobOnLastGrab();
                    mParentWindow.mainView.RefreshShotTab(actIndex);  //260406 hbk -- UI 갱신
                    return;
                }
            }
        }
    }

    //260406 hbk -- SimulImagePath 없으면 기존 시퀀스 실행 로직 유지
    if (treeListBox_sequence.SelectedItem is NodeViewModel) {
        // ... 기존 로직 그대로 유지 ...
    }
}
```

**핵심 분기:**
- Action 노드 선택 + Inspection 시퀀스 + SimulImagePath 존재 → `RunBlobOnLastGrab()` 직접 호출 (per D-04: 선택된 Shot만)
- 그 외 → 기존 `mParentWindow.StartSequence(seqID, actID)` 동작 유지

**주의사항 (Research Pitfall 1, 4, 5 반영):**
- `SystemHandler.Handle.Sequences.IsIdle` 체크 필수 (TCP 충돌 방지)
- 시퀀스 `Start()`를 호출하지 않음 (빈 시간폴더 생성 방지)
- `RunBlobOnLastGrab()`은 Step 상태에 의존하지 않으므로 리셋 불필요

**RefreshShotTab 호출:** `mParentWindow.mainView`에 `RefreshShotTab(int shotIndex)` 메서드가 있다면 호출. 없다면 `mParentWindow.mainView.SetParam` 패턴을 사용하여 해당 ShotTabView를 갱신한다. 실제 메서드명은 코드를 읽고 확인할 것.
