**RecipeFileHelper.cs 수정:**

`GetRecipeFilePath(string name)` 메서드 바로 아래에 다음 오버로드 2개를 추가한다 (기존 메서드는 절대 삭제하지 않는다):

```csharp
/// <summary>
/// Site별 레시피 파일 경로를 반환한다. (Recipe/SiteN/name/main.ini)
/// </summary>
public string GetRecipeFilePath(int siteNumber, string name) {
    string recipeSavePath = SystemHandler.Handle.Setting.RecipeSavePath;
    return System.IO.Path.Combine(recipeSavePath, "Site" + siteNumber, name, FILE_RECIPE + EXT_RECIPE);
}

/// <summary>
/// 지정한 Site의 레시피 목록을 수집하여 List를 갱신한다. (Recipe/SiteN/ 하위만 스캔)
/// </summary>
public int CollectRecipe(int siteNumber) {
    SystemHandler pSys = SystemHandler.Handle;
    string sitePath = System.IO.Path.Combine(pSys.Setting.RecipeSavePath, "Site" + siteNumber);
    List.Clear();
    if (System.IO.Directory.Exists(sitePath) == false) return 0;
    string[] recipeDirList = System.IO.Directory.GetDirectories(sitePath, "*");
    foreach (string recipeDir in recipeDirList) {
        string recipeName = new System.IO.DirectoryInfo(recipeDir).Name;
        string filePath = System.IO.Path.Combine(recipeDir, FILE_RECIPE + EXT_RECIPE);
        if (System.IO.File.Exists(filePath)) {
            System.DateTime creationDate = System.IO.File.GetCreationTime(filePath);
            System.DateTime lastOpenDate = System.IO.File.GetLastAccessTime(filePath);
            List.Add(new RecipeFileInfo { Name = recipeName, CreateDate = creationDate, LastOpenDate = lastOpenDate, FilePath = filePath });
        }
    }
    return List.Count;
}
```

주의:
- `FILE_RECIPE`와 `EXT_RECIPE`는 클래스 내 `private const`이므로 직접 참조 가능 (`"main"`, `".ini"`)
- `using System.IO`가 이미 파일 상단에 존재하므로 `System.IO.` 접두사 없이 `Path.Combine` 사용 가능. 단, 혼란을 피하기 위해 확인 후 적절히 사용
- `HasRecipe(string name)` 등 기존 메서드 일절 수정하지 않는다
