## Interaction Contract

### SettingWindow — ImageSavePath Row

- **Input type:** Text box with `[DirectoryPath]` attribute. PropertyTools.Wpf renders a folder-browse button automatically.
- **Default value displayed:** `D:\Log` (after Phase 11 code change; existing Setting.ini overrides for existing installs).
- **Validation:** PropertyTools.Wpf `[DirectoryPath]` provides built-in path validation. No additional validation logic required.
- **Tab interaction:** PropertyGrid row renders inside `Inspection | Image Save` category tab — no tab change needed.

### SettingWindow — SaveOkImage Checkbox

- **Default state:** Unchecked (false) — renders as standard WPF CheckBox.
- **Behavior on check:** Enables OK image saving (original + annotated pair). Takes effect on next inspection cycle.
- **No confirmation required:** Toggle is non-destructive.

### Image Save Feedback (no UI — log only)

- **Save success:** No UI feedback. Silent background operation.
- **Save failure:** `Logging.PrintLog(ELogType.Error, "SaveImage Error: {message}")` — appears in existing LogView, not as a dialog.
- **Rationale (source: RESEARCH.md Pattern 3):** Existing pattern uses try/catch + log. Phase 11 preserves this pattern. No MessageBox or status bar update.

|-------|-------|---------|--------|

No loading, skeleton, or progress states are needed. Phase 11 save operations are fire-and-forget background tasks (Task.Factory.StartNew pattern).
